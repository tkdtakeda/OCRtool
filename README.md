# 帳票OCR統合ツール (OCRtool)

罫線除去ツールと画像マッチングツールを統合し、**帳票の自動判定 → 傾き補正 → 罫線除去 → 領域OCR** を
ブラウザ上（サーバー不要）で行う認知心理学ベースのツールです。登録した帳票レイアウトと認識結果は
**IndexedDB** に永続化されます。

> ブラウザだけで動作します。OpenCV.js / Tesseract.js を CDN から読み込むため、初回はインターネット接続が必要です。

---

## 画面（2モード）

ヘッダーのトグルで 2 つの工程を切り替えます（選択負荷を下げる **Hick の法則**）。

### ① レイアウト登録工程 — `index.html`
1. **帳票名**と**基準画像**（帳票全体）を設定
2. 基準画像の上に**識別アンカー**（複数可）を矩形で描画 … 帳票判定・傾き補正に使用
3. 同じ画像に**OCR領域**を矩形で描画 … 基準画像の**絶対座標**で保存
4. **OCR(PSM)・罫線除去パラメータ**を設定（既定値は罫線除去ツールから引き継ぎ）
5. 保存 → **IndexedDB** に永続化（再読込後も保持）

> 識別アンカーは「基準画像上にドラッグ」または「別画像を貼り付け→基準画像内を自動検出して配置」の
> 2 通りで追加できます。これにより「特徴画像を複数登録する」要件を満たします。

### ② OCR実行工程 — `index.html`
1. 帳票画像を読み込む（**将来 PDF 対応**。現状は画像）
2. 登録帳票の**全アンカー**と一括照合し、スコア順にランキング
3. 帳票を**自動判定**（下記アルゴリズム）。低確信は「要確認」で手動選択に回す
4. 登録画像と比較して**傾きを補正**
5. 補正後に**罫線除去**（登録パラメータを引き継ぎ）
6. 登録した**OCR領域**を認識し、**信頼度つきで IndexedDB に保存・表示**（履歴一覧）

#### 認識設定の調整（デバッグ）・プリセット・全PSM比較
OCR実行画面の「認識設定（デバッグ）」パネルで、**OCR言語・PSM・罫線除去パラメータ（膨張を含む全項目）**を
表示・変更し、「この設定で再実行」で**再マッチング無しに**即試行できます。

- **プリセット**: 良い値が決まったら「現在値を保存」で**名前付きプリセット**として IndexedDB に保存。
  以後はプリセットを選んで「適用」→「再実行」で素早く切替できます（登録画面に戻る必要なし）。
  「設定を帳票に保存」で帳票自体へ反映も可能。
- **全PSM比較**: 同じ罫線除去結果に対し、各 PSM（自動/単一行/単一単語/単一文字/疎/生1行 など）の
  読み取り結果と信頼度を**一覧比較**。最良を「採用」してワンクリックで反映できます。
- 罫線除去後プレビューは**拡大（ズーム）・ドラッグ移動（パン）**に対応。

#### 日本語OCR
Tesseract の言語は **英語 / 日本語＋英語 / 日本語** から選択できます（登録・デバッグ両方）。
既定は英語のため、日本語ラベルを含む領域は「日本語＋英語」を選択してください
（初回は言語データ `jpn` の DL に時間がかかります）。

**数字が丸数字（①②⑨）や漢数字（一）になる場合**：セル枠＋日本語モデルが原因です。OCR設定の
**文字ホワイトリスト**（`tessedit_char_whitelist`、例 `0123456789`）で字種を限定し、
**正規化**（①→1・全角→半角、既定ON）を併用すると確実に算用数字になります。
「1」が漢字の「一」になる（下の横棒＝罫線残りが原因）場合は、**水平罫線除去を強める**か、
**「漢数字→数字」トグル**（一→1、既定OFF・氏名欄では注意）をON、または数字欄の**言語を英語**にしてください。

**抽出パターン（正規表現）**：各OCR領域に書式（例 `[A-Z]{2}\d{4}`）を指定すると、はみ出した文字を
無視して該当部分だけを取り出します（例「注文No AK1029 控」→「AK1029」）。固定接頭辞＋固定桁数の
番号欄に有効で、登録画面のOCR領域リストで設定・保存できます。元の生テキストは結果欄に「元:」で併記。

**複数アンカー補正（スケール＋位置ずれ）**：1帳票に識別アンカーを2つ以上登録すると、認識時に
各アンカーの一致位置（中心）から **相似変換（拡大率＋平行移動）** を最小二乗推定し、OCR領域座標へ反映します。
スキャン倍率の違いや位置ずれを吸収し、領域外し・隣接文字の混入を軽減します（1点のみ一致時は平行移動のみ）。
さらに ①OCR領域から日本語ラベル（例「注文No」）を除外、②垂直罫線パラメータでセル区切り線を除去、
③PSM を「単一行/単一単語」に、④数字欄は言語を英語に、も有効です。
ホワイトリスト・正規化・言語・PSM・罫線除去は**プリセット**として保存・呼び出しできます。
識別アンカー描画は「移動」ツールで拡大後にパンできます。

単体ツールも残しています（ヘッダーから遷移）。

- `line_removal.html` … 罫線除去ツール（単体）
- `image_matcher.html` … 画像マッチングツール（単体）

## 帳票自動判定アルゴリズム（採用根拠）

ご提案の判定基準は次のものでした:

> 上位5位のなかで上位3以内に同一帳票が2つ以上、かつ全体で半数以上同一帳票であれば採用

これは**順位（ランク）のみ**を見るため、次の弱点があります。

1. **スコアの絶対値を無視**する … 全アンカーが弱い一致（例: peak 0.42）でも「順位条件」を満たせば採用してしまう
2. **アンカー登録数の偏りに弱い** … アンカーを多く登録した帳票がランキング上位を占有し、不当に有利になる

帳票誤判定は**全フィールドの誤読**に直結するため、本ツールは既定で
**スコア加重 + 確信度 + 棄却(abstention)** 方式を採用しました（`voting.js`）。

- **peak**：帳票ごとの最良アンカースコアを主証拠とする
- **裏付け(corroboration)**：複数アンカーの一致は**小さな加点**に留め、登録数バイアスを排除
  （`agg = peak × (1 + β × (裏付け数−1))`, β=0.06）
- **margin**：1位と2位の差で接戦かを評価
- **確信度** = 絶対スコアと margin の加重 → **採用 / 要確認 / 不一致** に分類
  - `peak < 0.45` → **不一致**（未登録帳票の可能性）
  - `確信度 ≥ 0.70 かつ margin ≥ 0.06` → **採用**
  - それ以外 → **要確認**（人手確認に回す）

ご提案の順位ルールの合否も「**補助指標**」として UI に併記し、判断の透明性を確保しています。
閾値は `voting.js` の `defaults()` で調整できます。

### 検証例（`voting.js` 単体）
| ケース | 提案ルール | 本方式 | 妥当性 |
|---|---|---|---|
| Bが単独で強い(0.93) vs Aが中庸2件(0.55) | B不合致 | **B採用** | 登録数バイアスを回避 |
| 順位ルール合致だが全体的に低スコア(peak0.42) | 合致→採用 | **不一致(棄却)** | ノイズ採用を防止 |
| 1位と2位が接戦 | 合致 | **要確認** | 人手確認に委譲 |

## 認知心理学に基づく設計

| 原則 | 実装 |
|---|---|
| **Hick の法則**（選択負荷） | 上位操作を「登録 / OCR」の 2 モードに限定 |
| **認識 > 想起** | 帳票・領域をサムネイルと色で提示。座標や名称を記憶させない |
| **システム状態の可視化**（Norman） | マッチング→判定→補正→除去→OCR の進捗を常時表示 |
| **直接操作** | OCR領域は画像上に直接描画。結果も同じ座標系で重畳表示 |
| **一貫した色分け**（Von Restorff） | 領域色を登録・プレビュー・結果で統一 |
| **段階的開示** | 罫線除去など詳細設定はアコーディオンに格納 |
| **クローズ欲求**（Zeigarnik） | 登録の完了チェックリストで未完了を可視化 |
| **信頼の較正** | 確信度と根拠（ランキング）を提示し、低確信時は人間に判断を委ねる |

## データモデル（IndexedDB: `chouhyou_ocr`）

```jsonc
// forms ストア（登録帳票）
{
  "id": "…",
  "name": "発注書",
  "referenceImage": { "dataURL": "…", "w": 640, "h": 760 },   // 帳票全体（OCR座標の基準）
  "anchors": [                                                 // 識別用特徴画像（複数）
    { "id":"…", "name":"タイトル", "dataURL":"…", "w":200, "h":48, "refX":30, "refY":20 }
  ],
  "ocrRegions": [                                              // 基準画像の絶対座標
    { "id":"…", "name":"番号", "x":105, "y":78, "w":170, "h":18 }
  ],
  "ocrSettings": { "psm": 7, "lang": "eng" },
  "lineRemoval": { /* 罫線除去パラメータ（既定は罫線除去ツールから引き継ぎ） */ }
}

// results ストア（認識結果・信頼度つき）
{
  "id":"…", "formId":"…", "formName":"発注書", "createdAt": 0,
  "decision":"accepted", "confidence":0.92, "angle":-1.5,
  "fields":[ { "name":"番号", "text":"PO-2024-0001", "confidence":96 } ],
  "voting":{ "margin":0.7, "ranking":[…], "legacySignal":{…} }
}
```

座標変換：認識時、採用アンカーを回転補正後画像へ再マッチして平行移動量
`T = (一致位置) − (anchor.refX, refY)` を求め、OCR領域の基準座標 `(x,y)` を `(x+T.x, y+T.y)` に写します。
これにより**絶対座標**と**複数アンカー**の両立、スキャンずれの吸収を実現しています。

## 使い方（`file://` で直接開く）

サーバーは不要です。**`index.html` をダブルクリック**してブラウザで開くだけで動作します。
すべてのスクリプトは相対パスのローカル読み込み（ES モジュール不使用）なので `file://` で問題なく動きます。
登録帳票・認識履歴は `file://` オリジンの **IndexedDB** に保存され、再読込後も保持されます（実機確認済み）。

1. 「① レイアウト登録」で「サンプル登録」を押す（または新規に帳票を登録）
2. 「② OCR実行」に切替 →「サンプル画像（傾き付）」→「認識を実行」
3. 判定パネルで帳票と確信度を確認 → OCR 結果と履歴（IndexedDB）を確認

> **注意**
> - 初回のみ OpenCV.js / Tesseract.js を CDN から取得するため**インターネット接続**が必要です
>   （アプリ本体・登録・履歴はローカルのみで動作）。
> - **Chrome / Edge / Firefox 推奨**。Safari は `file://` での IndexedDB を制限するため、
>   保存が無効になる場合があります（その旨を画面に通知します）。
> - どうしても従来どおりサーバー経由で開きたい場合は `python3 -m http.server 8000` 等も利用できます。

## 今後の拡張（PDF対応）

`studio_app.js` の `fileToCanvas(file)` がファイル種別の分岐点です。`application/pdf` の分岐に
pdf.js を組み込み、ページを Canvas に描画して既存パイプラインへ渡すことで対応できます
（データモデル・座標系はそのまま流用可能）。
